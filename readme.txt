Необходимо установить:
  pip install selenium
  pip install torch
  pip install peft
  visual c++ driver

Краткое руководство по директориям:
  database/black_account_data.txt - содержит все 'черные' аккаунты в формате loggin,password,cookie,proxy
  database/break_bot.txt - содержит все аккаунты которые сломались во время работы, либо при запуске в формате loggin,password,proxy
  database/generation_busy.txt - очередь на генерацию ответов, так как нейрогенерация довольно ресурсозатратная,
                                имеется ограничение на количество ботов которым разрешенно в один момент времени её использовать 
  database/grey_account_data.txt - содержит все 'серые' аккаунты в формате loggin,password,cookie,proxy (пока не используется, нет серых аккаунтов)
  database/human_list.txt - список ссылок на аккаунты, прошедшие проверку на 'реальных людей', сюда попадают аккаунт прошедшие проверку соотношения подписок к подписчикам
                            (подписки/подсписчики < 2 && подсписчики/подписки <=4), то есть пользователи у которых примерное рановное отношение числа подписок к подписчикам
  database/subscription_list.txt - список всех аккаунтов на которые мы подписываемся в группах 
  database/titles_group.json - файл с названиями групп, в которых мы подписываемся на пользователей (можно было просто ссылками реализовать в текстовом как и прошлые) 

  generation_config - конфиги для генерации нерокоменнтов

  image - изображения которые вставляем во все посты для выделения наших комментов или постов

  refactor_database - вспомогательная папка содержащая неотформатированные файлы с данными от 'черных' или 'серых аккаунтов,
   а также файл с прокси и скрипт который быстро раскидывает все по нужным файлам

  webdriver - драйвер для работы selenium от хрома версии 117.0.5938. ..

  bot_sample - основной исполняемый файл для запуска ботов

  generation_reply  - скрипт с нерокоменнтариями, получает на вход текст и на его основе формирует комментарий, опираясь на конфиги из папки generation_config

Руководство по основному скрипту:
  Вначале указывается путь до файла с данными от 'черных' аккаунтов и до json файла с названиями групп, текст который мы хотим запостить на стену и путь до драйвера.
В точке запуска программы получаем массив с аккаунтами и через мультипроцессинг с таймаутов в 15 секунд запускаем все аккаунты. Запуск происходит путем открытия твитера
и внесения куки от нужного файла, в случае ошибки, на протяжении всей работы бота, в файл break_bot сохраниться логин пароль и прокси от аккаунта, чтобы можно было проверить в ручную или перезапустить.

  После открытия аккаунта попадаем в функцию main, в которой мы создаем пост с текстом описанным вначале и случайной картинкой из папки image (пока есть неполадки с функцией, но в целом рабочая), 
затем переходим в группы и подписываемся на пользователей, краткая логика следующая:
  1) выбираем случайное количество пользователей на которое хотим подписаться
  2) проходим по порядку по всем группам из json файла
  3) в каждой группе берем от 1 до 5 пользователей и подписываемся на них параллельно записывая ссылки этих пользователей в subscription_list (так удобнее потом проходить по ним)
  4) делаем пока не закончаться действия из первого пункта.

Когда кончаются действия попадаем в следующий метод с обходом subscription_list:
  1) также выбираем случайное число аккаунтов на которые хотим зайти
  2) получаем из файла случайную ссылку
  3) переходим по этой ссылке и проверяем является ли аккаунт ботом
  4) если не бот, то также проверяем является человеком и если да то заносим ссылку в human_list
  5) если аккаунт не бот делаем комментарий на последний пост + лайкаем его (как флажок, чтобы не комментить один и тот же пост)
  6) делаем пока не кочаться действия

Также файл содержит реализацию и других методов например: репост, репост с ответом, поиск по хештегу. Но эти функции пока не требовались, в случае чего можно будет их интегрировать в реализацию.
